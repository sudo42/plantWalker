about this software

what is it?

I call this software "don't walk on the plants". It is based on the concept of minesweeper, but uses plants instead of bombs, because plants are much prettier and way more peaceful.

how to use it?

You see the little input boxes at the top of the page? The first pair is to specify the width and height of the grid. The one below them is for the number of flowers to place on that grid.
Once you have entered some values (or just leave the defaults in there) click the "generate" button. A grid should appear.

Use single (primary) click on one of the cells, to explore it. With your first click, all flowers will be placed. Using a secondary click (the other mouse button, and usually long-pressing on touch screens), or pressing at least one of shift, ctrl, alt while clicking, will toggle a flag on any unexplored cell. Flags prevent accidentally exploring a cell, and are usually used to mark where you belive flowers are located.

Some details: The time will start with your first click, and stop when you stepped on a flower or successfully explored all all ground withot flowers.
Minimum value for width and height is 1, although I would recommend using at least 12 x 6. (The visuals might get funny for to low values. In additon: you probably want to stay below 90 or so, as too high values might have a severe performance impact, and a huge grid is probably annoying to play on. ) You can have 0 flowers, sure, but not less, and at most as many as cells on the grid. (Although I would advise against too extreme numbers.)
The "generate" button should get a red shadow and be disabled when impossible values are entered. (At least for the most obvious logic problems, negative numbers, etc. â€¦ regarding overflows, prcision limits, and similar, please consider my advise against too high values.)

requirements and technical stuff?

This little "game" runs entirely in your browser; You can even download the files and play it offline from your mass storage device.

There are no external libraries, artwork, scripts, â€¦ or any reason to perform additional requests for anything which are not the project files. At the moment, i use emoji instead of images, so you probably need a somewhat decent emoji font installed and enabled (dont't worry, this should be the case for most devices nowadays).

Your browser defaults are used for the font, and "basic" text colours. (As the game itself has some coloured elements it wasn't easily possible to retain default colours there.) This has to advantages: It is possible that you used some non-standard font and font-size for some reasons (personal preference, dealing with dyslexia, too tiny text by default, â€¦), and in that case: Why should I code against your wished for no good reason? The second advantage: I didn't have to think about it that much.
The above also implies: Should you desire a "dark mode" for this page (at least the outer parts), just change your browser defaults.

For your browser â€¦ use a recent version of something good, and enable JavaScript. (No years old ESR versions, no iexplore.exe, ... probably not Egde or Safari, but you might be lucky.) I personally only tested it in recent (beta and developer) versions of Firefox for Linux and Android. (On that note: this application is in no way optimised for mobile devices ... so don't expect any decent user experience.)
